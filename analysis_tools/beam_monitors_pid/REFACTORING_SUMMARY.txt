## Module Breakdown

### 1. constants.py (120 lines)
**Purpose:** Centralized detector configuration and physical constants

**Contains:**
- Physical constants (speed of light: c = 0.299792458 m/ns)
- Detector distances (L, L_t0t4, L_t4t1)
- Particle masses (Electrons, Muons, Pions, Protons)
- TOF cut values for particle identification  
- TDC channel groups (T0, T1, T4, T5 detector channels)
- ACT (Anti-Coincidence Trigger) groups
- HC (Hadronic Calorimeter) settings
- Channel name mappings for 12 ACT detector channels

**Key Constants:**
- `c`: Speed of light (0.299792458 m/ns)
- `particle_masses`: Dict of particle masses in GeV/c²
- `t0_group`, `t1_group`, `t4_group`: Detector channel lists
- `t5_total_group`: 8 bar pairs for TOF detector
- `CHANNEL_MAPPING`: PMT ID → human-readable names

### 2. file_utils.py (133 lines)
**Purpose:** File I/O operations for EOS storage access

**Functions:**
1. `stage_local(src_eos_path, min_free_gb=20, min_bytes=1_000_000)` → str
   - Stages ROOT files from CERN's EOS to local /tmp
   - Checks disk space (default 20 GB minimum)
   - Uses hash-based naming to avoid collisions
   - Returns empty string if insufficient space

2. `to_xrootd(path)` → str
   - Converts EOS path to XRootD URL format
   - Used by xrdcp for remote file copying
   
3. `make_blocks(idx, max_block)` → list[tuple]
   - Partitions index arrays into contiguous blocks
   - Enables efficient block-wise reading
   - Returns list of (start, end) tuples

### 3. flag_utils.py (93 lines)
**Purpose:** Bitmask-based event quality flag management

**Key Insight:** Multiple boolean flags packed into single integer for storage efficiency

**Functions:**
1. `make_flag_map(flags)` → dict
   - Creates deterministic flag-to-bit mapping
   - Uses sorted order for reproducibility

2. `write_event_quality_mask(flag_dict, flag_map)` → int
   - Packs boolean flags into integer bitmask
   - Each flag gets one bit position
   - Missing flags treated as False

3. `read_event_quality_mask(mask, flag_map)` → dict
   - Reverses the packing operation
   - Recovers original flag dictionary from bitmask

**Tracked Flags:**
- event_q_t0_or_t1_missing_tdc
- event_q_t4_missing_tdc
- event_q_t5_missing_tdc
- event_q_hc_hit
- event_q_t4_missing_qdc
- t5_more_than_one_hit

### 4. detector_utils.py (120 lines)
**Purpose:** TDC hit processing and detector requirement validation

**Key Problem:** TDC channels sometimes record multiple hits (shouldn't happen)

**Functions:**
1. `deduplicate_tdc_hits(ids, times)` → (ids_clean, times_clean, duplicates_removed)
   - Removes duplicate TDC hits per channel
   - Keeps only first (earliest) hit
   - Returns count of removed duplicates per channel
   - Raises `NotTheFirstHit` if hits not ordered correctly

2. `tdc_requirement_met(group, tdc_set)` → bool
   - Validates detector group requirements
   - Supports two modes:
     - "all": All channels must be present
     - "any_pair": At least one channel pair must be complete

**Exception:**
- `NotTheFirstHit`: Raised when TDC hits are improperly ordered

### 5. fitting.py (214 lines)
**Purpose:** Curve fitting functions for detector response analysis

**Curve Functions:**
1. `gaussian(x, amp, mean, sigma)` → float/array
   - Standard Gaussian/normal distribution
   - Parameters: amplitude, mean, standard deviation

2. `three_gaussians(x, amp, mean, sigma, amp1, mean1, sigma1, amp2, mean2, sigma2)` → array
   - Sum of three Gaussians
   - Used for fitting multi-peak distributions (e.g., T4 TOF)

3. `landau_gauss_convolution(x, amp, mpv, eta, sigma)` → array
   - Convolution of Landau (energy loss) and Gaussian (detector resolution)
   - Uses numerical integration with adaptive domain
   - Includes error handling for numerical overflow

**Fitting Functions:**
1. `fit_gaussian(entries, bin_centers)` → (popt, pcov)
   - Fits single Gaussian to histogram
   - Auto-generates initial guesses from histogram statistics
   - Returns optimized parameters and covariance matrix

2. `fit_three_gaussians(entries, bin_centers)` → (popt, pcov)
   - Fits three Gaussian peaks
   - Secondary peaks offset ±4 bins from primary
   - Useful for multimodal distributions
